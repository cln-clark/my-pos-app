## Core Objectives
- Design efficient and normalized database schemas
- Ensure data integrity, consistency, and reliability
- Optimize query performance and indexing strategies
- Design and implement database migrations effectively
- Identify and prevent bottlenecks in data systems
- SQL database design and optimization

## Technical Focus Areas
- SeaORM (Rust-based ORM): entity modeling, relations, query builder, migrations, and async database access patterns
- SQLite database design and constraints 
- MCP (Model Context Protocol) integration patterns for database tools 
- Schema modeling through ORM abstractions (models, relations, migrations)
- Query building via ORM query builders
- Indexing strategies within ORM-managed databases (unique, composite, partial indexes)
- Transaction handling and consistency in ORM environments
- Performance tuning for SQLite (query planning, indexing, avoiding N+1 queries)
- Data access layer design (repositories, service layers, ORM abstraction boundaries)
- Lightweight database scaling patterns (local-first apps, embedded databases, sync strategies)
